# Ansible Implementation Plan - Travel Management System

## Overview
Automate the entire Travel Management System infrastructure using Ansible playbooks to deploy Docker Compose services, databases, monitoring, and security components on a single local server.

---

## Phase 1: Ansible Project Structure Setup

### Directory Structure
```
ansible/
├── inventory/
│   └── inventory.ini           # Local host inventory
├── group_vars/
```

## Phase 2: Core Roles Implementation

### 2.1 Docker Role
**Purpose**: Ensure Docker and Docker Compose are installed and running

**Tasks**:
- Install Docker CE
- Install Docker Compose plugin
- Add current user to docker group
- Start and enable Docker service

**Key Modules**: `apt`, `systemd`, `user`, `group`

---

### 2.2 Databases Role
**Purpose**: Deploy PostgreSQL and Neo4j using Docker Compose

**Tasks**:
- Create necessary directories for data persistence
- Deploy PostgreSQL container from db/docker-compose.yml
- Deploy Neo4j container from db/docker-compose.yml
- Configure environment variables
- Wait for databases to be ready
- Initialize database schemas (via Flyway migrations)

**Key Modules**: `community.docker.docker_compose_v2`, `community.docker.docker_container`, `wait_for`

**Files**: `db/docker-compose.yml`, `files/init-schemas.sql`

---

### 2.3 Microservices Role
**Purpose**: Build and deploy all Spring Boot microservices

**Tasks**:
- Build Docker images for each service (eureka, apigateway, user, travel, payment)
- Configure environment variables (Eureka URL, database connections)
- Deploy services using docker-compose
- Configure health checks
- Wait for Eureka Server first
- Deploy services in dependency order

**Key Modules**: `community.docker.docker_image`, `community.docker.docker_compose_v2`, `uri`

**Services**:
1. eureka-service (port 8761)
2. apigateway-service (port 8443)
3. user-service (internal)
4. travel-service (internal)
5. payment-service (internal)

**Template Variables**:
```yaml
microservices:
  eureka:
    port: 8761
  gateway:
    port: 8443
  services:
    user: {}
    travel: {}
    payment: {}
```

---

### 2.4 Frontend Role
**Purpose**: Deploy Angular Admin Dashboard

**Tasks**:
- Build Angular production bundle
- Create Nginx-based Docker image
- Deploy frontend container
- Configure reverse proxy to API Gateway

**Key Modules**: `shell`, `community.docker.docker_image`, `community.docker.docker_compose_v2`

---

### 2.5 Vault Role
**Purpose**: Deploy and configure HashiCorp Vault for secrets management

**Tasks**:
- Deploy Vault container
- Initialize Vault (if not initialized)
- Unseal Vault using pre-defined keys
- Enable KV secrets engine
- Store database credentials
- Store JWT secrets
- Store Stripe/PayPal API keys
- Configure Vault authentication method for containers

**Key Modules**: `community.docker.docker_container`, `uri`, `hashi_vault` lookup plugin

**Configuration**:
```yaml
vault:
  version: "1.15.0"
  port: 8200
  secrets:
    path: "secret/data/travel-system"
    keys:
      - "postgres_password"
      - "neo4j_password"
      - "jwt_secret"
      - "stripe_api_key"
      - "paypal_client_id"
```

---

### 2.6 Logging Role (Loki + Promtail)
**Purpose**: Centralized log aggregation

**Tasks**:
- Deploy Loki server container
- Deploy Promtail agent
- Configure Promtail to scrape logs from all Docker containers
- Configure Loki retention policies
- Deploy Grafana for log visualization

**Key Modules**: `community.docker.docker_compose_v2`, `template`

**Configuration**:
```yaml
logging:
  loki:
    port: 3100
    retention: "30d"
  promtail:
    port: 9080
    scrape_configs:
      - job_name: "docker-containers"
        docker_sd_configs:
          - host: "unix:///var/run/docker.sock"
```

---

### 2.7 SSL Role
**Purpose**: Manage SSL/TLS certificates

**Tasks**:
- Copy existing certificates from files/certs to target
- Set correct permissions on certificates
- Configure Docker volumes to mount certificates
- Update docker-compose templates to use certificates
- Verify certificate validity

**Key Modules**: `copy`, `file`, `stat`

**File Structure**:
```
files/certs/
├── server.crt
└── server.key
```

---

### 2.8 Jenkins Role
**Purpose**: Configure Jenkins with Ansible integration

**Tasks**:
- Deploy Jenkins container
- Install required plugins (Ansible, Docker, Git)
- Configure Jenkins credentials for Ansible
- Update Jenkinsfile to integrate Ansible playbooks
- Create Jenkins jobs for each deployment stage

**Key Modules**: `community.docker.docker_compose_v2`, `uri`

**Jenkins Pipeline Integration**:
```groovy
stage('Deploy with Ansible') {
    steps {
        sh 'ansible-playbook ansible/playbooks/deploy_services.yml'
    }
}
```

---

### 2.9 SonarQube Role
**Purpose**: Code quality analysis

**Tasks**:
- Deploy SonarQube container
- Deploy PostgreSQL for SonarQube database
- Configure SonarQube with quality profiles
- Integrate with Jenkins pipeline

**Key Modules**: `community.docker.docker_compose_v2`, `uri`

**Jenkins Integration**:
```groovy
stage('SonarQube Analysis') {
    steps {
        sh './mvnw sonar:sonar'
    }
}
```

## Phase 3: Playbooks

### 3.1 Master Playbook (site.yml)
```yaml
---
- name: Deploy Complete Travel Management System
  hosts: localhost
  connection: local
  become: true
  
  roles:
    - docker
    - databases
    - ssl
    - vault
    - microservices
    - frontend
    - logging
    - jenkins
    - sonarqube
```

## Phase 9: Testing Strategy

### Unit Testing Each Role:
```bash
ansible-playbook roles/docker/tasks/main.yml --check
ansible-playbook roles/databases/tasks/main.yml --check
```

### Integration Testing:
```bash
ansible-playbook ansible/playbooks/site.yml --check --diff
```

### End-to-End Testing:
1. Run complete deployment playbook
2. Verify all containers running: `docker ps`
3. Check service health: `curl http://localhost:8761` (Eureka)
4. Test API Gateway: `curl https://localhost:8443`
5. Verify logs in Grafana
6. Trigger Jenkins pipeline
7. Test rollback functionality

## Summary

This plan provides a complete Ansible automation framework that:
- ✅ Uses Docker Compose for all services (as per requirements)
- ✅ Deploys PostgreSQL and Neo4j databases
- ✅ Manages secrets with HashiCorp Vault
- ✅ Implements logging with Loki + Promtail
- ✅ Integrates with Jenkins for CI/CD
- ✅ Includes SonarQube for code quality
- ✅ Uses existing SSL certificates
- ✅ Deploys on single local server
- ✅ Follows Ansible best practices (roles, idempotent tasks)
- ✅ Supports load balancing (via Docker Compose replicas)
- ✅ Provides rollback mechanism
- ❌ No Kubernetes (as requested)
- ❌ No bonus features (as requested)
